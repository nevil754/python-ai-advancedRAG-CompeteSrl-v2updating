
########REPRODUCE AL 100% ESATTAMENTE CON STESSE VERSIONS PLUGINGS installate (anche con quelle TRANSITIVE!!):
devi assicurarti di bloccare le dipendenze transitive (quelle che anche se non scarichi direttamente, vengono scaricate).

1.crea ambiente asettico
conda create -n envCompeteSrlRAGproductionOKpluginsTransitivi python=3.11.9  #USO VERSION 3.11.9🔥
conda activate envCompeteSrlRAGproductionOKpluginsTransitivi
python -m pip install --upgrade pip
pip install pip-tools

2.assicurati che i file dove hai elencato tutti i plugins, non si chiamino .txt --> .in ! 
3.genera i lock file con versioni transitive bloccate runnando (in root prj):
pip-compile requirements.in --output-file requirements.txt --resolver=backtracking --no-header --annotate
//prima avevo anche '--pre' finale per gestire la versione alpha di markitdown, ma ora faccio direttamente >=0.0.1
pip-compile requirements-dev.in --output-file requirements-dev.txt --resolver=backtracking --no-header --annotate

ora hai appena creato i files .txt, con ogni singolo pacchetto (diretto e TRANSITIVO) pinato alla versione esatta!!
INFATTI SE GUARDI IL FILE .TXT GENERATO VEDRAI ANCHE LE DIPENDENZE TRANSITIVE CON LE LORO VERSIONI!! quindi ora se fai 'pip install..' su quello, pip non decide piu come vuole lui le dipendenze transitive!!
4. assicurati di aggiungere '--no-deps' nei Dockerfile (e.g.celery.Dockerfile) quando fai ... --no-cache-dir --no-deps -r requirements.txt
5. solo ORA puoi installare i pacchetti in modo deterministico
pip install -r requirements.txt
pip install -r requirements-dev.txt
//tutto è già pinato — pip non ha margine per "risolvere"(CIOE CHE DECIDE LUI) niente!! ORA PRJ E' RIPRODUCIBILE AL 100% ovunque senza che ci siano bug che le dipendenze transitive le versioni sono cambiate!!
//ora non hai piu bug silenziosi causati dalle incostistenze delle versions dei pacchetti transitivi, tutto il prj con pacchetti diretti&transitivi E RIPRODUCIBILE ESATTAMENTE UGUALE su qualsiasi macchina.

//è ok, xk ricorda che in produzione Docker ogni container è già un ambiente isolato!
